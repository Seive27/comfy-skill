# Comfy Skill

<p align="center">
  <em>The Anti-Slop Frontend Framework for AI Agents</em>
</p>

Comfy Skill helps AI agents build better interfaces by transforming prompts into structured frontend decisions.

Instead of generating arbitrary layouts, Comfy Skill introduces:

* Taste Profiles
* Skill Graphs
* Deterministic Rendering
* Anti-Slop Audits
* Consistent Design Systems

The goal is simple:

**Less AI slop. More intentional interfaces.**

---

## Installation

Install Comfy Skill using Agent Skills:

```bash
npx skills add https://github.com/Seive27/comfy-skill
```

Install the main frontend skill directly:

```bash
npx skills add https://github.com/Seive27/comfy-skill --skill "design-comfy-frontend"
```

---

## Quick Start

Load Comfy Skill at the beginning of your session:

```text
I have loaded Comfy Skill v1 as my only source of frontend design rules.
```

Then provide your project brief.

Example:

```text
Brief:
- Page kind: Landing Page
- Product: Comfy Skill
- Audience: AI engineers and developers
- Brand vibe: playful, structured, modern
- References: Linear, Raycast, Arc Browser
- Avoid: generic AI SaaS designs
```

## Core Principles

### Taste Profiles

Every project begins with a Taste Profile.

A Taste Profile defines:

* Visual personality
* Layout direction
* Information density
* Motion intensity
* Interaction style

---

### Skill Graphs

Comfy Skill converts design decisions into structured skills.

Example:

```text
Hero
 ↓
Feature Grid
 ↓
Architecture Section
 ↓
CTA
```

Instead of random generation, interfaces are built through composable design skills.

---

### Deterministic Rendering

The same brief should produce the same design direction.

Comfy Skill favors:

* consistency
* predictability
* maintainability

over randomness.

---

### Anti-Slop Audits

Every output is checked for:

* repetitive layouts
* generic SaaS patterns
* poor hierarchy
* weak visual structure
* inconsistent spacing
* unnecessary filler content

## Why Comfy Skill?

Most AI-generated websites suffer from:

* repetitive layouts
* generic design patterns
* weak hierarchy
* placeholder content
* visual inconsistency

Comfy Skill introduces structure before generation.

The result is more intentional, maintainable, and recognizable interfaces.
